Setting up the DB2 Health Monitor Sample Application for PHP


There are five main steps required to set up the sample application:

  1. Install a DB2 9 server product and create the DB2 sample database.
  2. Install and configure PHP.
  3. Install and configure the sample application.
  4. Test the database connection.
  5. Test the Web service.

Creating the DB2 Sample Database

You can create the DB2 sample database either:

  • Locally - On the same machine as the sample application.
  • Remotely - On a different machine than the sample application.

Installing PHP

PHP can be manually installed and re-built with the DB2 libraries. Alternatively, download and install PHP Zend Core for IBM with pre-configured DB2 server support.

If PHP has been manually compiled with DB2 server support you must configure the ibm_db2 extension in order for the sample to work. Detailed instructions are listed in the Setup sections.

Similarly, you must also configure the SOAP extension in order for the sample to work. Detailed instructions are listed in the Setup sections.

Configuring the Sample Application

After you install and configure the DB2 9 server, sample database, HTTP server, and PHP, configure the sample application.

The Web Service Consumer features an XML configuration file in which the Web Service Producer WSDL document address can be set without having to change any PHP code.

Similarly, you must set the address of the SOAP server in the WSDL file (part of the Web Service Producer).

After this, all you need to do is drag and drop the DB2 Health Monitor Sample Application into the publicly visible folder used by the Apache HTTP Server.

Testing the Connection

After everything is set up, test the connection to the DB2 sample database. If the connection succeeds, leave it active to ensure that health monitoring data is being gathered on a regular basis.

Testing the Service

Finally, ensure that the service is functioning properly. This involves three steps:

  1. Ensure that the Web service WSDL document is accessible through the HTTP server.
  2. Ensure that the Web service SOAP server is accessible via the URL in the WSDL document.
  3. Ensure that the Web Service Consumer GUI displays health monitoring data.
